# Backend — WebAuthn Device Registration POC

FastAPI backend providing WebAuthn registration and authentication APIs with
**Invitation Token** based device onboarding (no username/password).

## Setup

```bash
pip install -r requirements.txt
python -m uvicorn main:app --reload --port 8000
```

> Note: `uvicorn` may not be on PATH — use `python -m uvicorn` as shown.

## API Documentation

Once running, visit:
- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

## In-Memory Data

No database required. The `models/device.py` file contains a pre-seeded
dictionary of 10 employee/device records keyed by `employee_id` (e.g., `EMP001`).
There are **no passwords** — onboarding is gated by single-use invitation tokens
that an administrator generates on demand. All data is lost on server restart.

## Invitation Tokens

- Generated by an admin (`POST /admin/generate-token`) — format `INV-XXXX-XXXX-XXXX`
- Single-use and time-limited (default 24h, configurable via `INVITATION_TOKEN_HOURS`)
- Validated by `POST /register/options` before any WebAuthn options are returned
- Automatically consumed (deleted) after a successful registration
- Can be revoked before use (`POST /admin/revoke-token`)

## Admin Endpoints

All admin endpoints require the `X-Admin-Key` header (POC key: `admin-secret-key`):

- `GET  /admin/devices`        — list all employee/device records
- `POST /admin/revoke`         — force-revoke a device (clears passkey + session)
- `POST /admin/generate-token` — issue an invitation token for an employee
- `POST /admin/revoke-token`   — revoke an unused invitation token
